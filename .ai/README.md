# AI Workspace

This directory contains AI context, memory, and work artifacts to help Claude Code and Zed maintain long-term project knowledge.

## Directory Structure

### `memory/`
**Purpose**: Long-term context that AI should always reference

Files:
- `decisions.md` - Architecture and design decisions
- `patterns.md` - Code patterns and idioms we use
- `conventions.md` - Team conventions and standards
- `pain-points.md` - Known issues and things to avoid

**When to update**: After major decisions or when establishing new patterns

### `tasks/`
**Purpose**: Active work and planning

Files:
- `current-prd.md` - What we're building right now
- `backlog.md` - Future features and improvements
- `bugs.md` - Known bugs and issues
- `questions.md` - Open questions needing answers

**When to update**: Daily, as work progresses

### `sessions/`
**Purpose**: Ralph loop session logs and transcripts

Files:
- Named by date and task: `YYYY-MM-DD-task-description.md`
- Auto-generated by Ralph loops
- Contains: task, iterations, outcome

**When to update**: Automatically after each Ralph loop

### `prompts/`
**Purpose**: Reusable prompt templates for common tasks

Files:
- `refactor-component.md` - Standard refactoring template
- `add-feature.md` - New feature template
- `fix-tests.md` - Test fixing template
- `add-docs.md` - Documentation template

**When to update**: When you create a prompt you'll reuse

### `archive/`
**Purpose**: Completed work kept for reference

Subdirectories:
- `completed-prds/` - Finished PRDs
- `old-sessions/` - Old Ralph sessions (older than 30 days)

**When to update**: Move completed items here to prevent clutter

## Usage Guidelines

### For Developers
1. **Before starting work**: Read `memory/` and relevant `tasks/`
2. **During work**: Update `tasks/current-prd.md` with progress
3. **After major decisions**: Document in `memory/decisions.md`
4. **When done**: Move completed PRDs to `archive/`

### For AI (Claude/Zed)
1. **Always read**: `memory/` files at session start
2. **Reference**: `tasks/current-prd.md` for current context
3. **Use**: `prompts/` templates for repetitive tasks
4. **Log**: Ralph sessions to `sessions/`
5. **Archive**: Completed work after 30 days

## Examples

### Starting a new feature
```bash
# Create PRD
cat > .ai/tasks/current-prd.md << 'EOF'
# Feature: Character Export
...
EOF

# Tell Claude
claude "Read .ai/tasks/current-prd.md and .ai/memory/patterns.md, then implement the character export feature"
```

### Running a Ralph loop
```bash
# Saves session to .ai/sessions/2026-01-10-add-tests.md
claude "/ralph-loop 'Add tests to untested components. Log session to .ai/sessions/$(date +%Y-%m-%d)-add-tests.md. Output <promise>COMPLETE</promise> when done.' --max-iterations 30"
```

### Recording a decision
```bash
# After deciding on state management
echo "## State Management Decision (2026-01-10)
- Use Tanstack Query ONLY for server state
- No Redux, Zustand, or Context for data fetching
- Rationale: Simpler, less boilerplate
" >> .ai/memory/decisions.md
```

## Maintenance

**Weekly**:
- Review `tasks/backlog.md`
- Update `memory/` if patterns changed

**Monthly**:
- Archive old sessions: `mv .ai/sessions/*.md .ai/archive/old-sessions/`
- Archive completed PRDs: `mv .ai/tasks/done-*.md .ai/archive/completed-prds/`
